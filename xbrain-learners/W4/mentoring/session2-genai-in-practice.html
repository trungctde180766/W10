The 4 Limitations of LLMs.

Why every GenAI pattern exists — and what it's really trying to work around.

01 · Foundations

Every LLM tends to have these 4 problems.

01
Hallucination
Confidently says wrong things. Predicts tokens that "sound right", not "are true". No source of truth built in.
02
Knowledge Cutoff
Only knows training data. Doesn't know anything that happened after. Frozen in time by design.
03
No Memory
Each request is independent. Doesn't "remember" previous turns. Every conversation starts from zero.
04
Can't Execute
Outputs text only. Can't run code, query databases, or call APIs. Can only suggest actions.
Every pattern in GenAI exists to solve one of these 4 limitations.

Patterns That Solve Each Limitation.

RAG, Tool Use, Agents, Fine-tuning, Guardrails — each is an answer to a specific problem.

02 · Pattern Map

The GenAI Pattern Map.

Limitation Pattern What It Does
Hallucination + Cutoff RAG Give model real documents before answering. Model weights unchanged.
Can't Execute Tool Use Model says "call function X" — your code executes it.
No Memory Context Engineering You decide what goes in the context window. Memory is your responsibility.
Complex Reasoning Agents / ReAct Think → Act → Observe → Repeat until goal is reached.
Wrong Domain / Voice Fine-tuning Train further on your own data. Actually changes model weights.
Unsafe Output Guardrails Filter input and output. Block harmful or off-topic content.
No pattern "fixes" the LLM — they all work around limitations.
02 · PatternsSession 2
02 · RAG

RAG: Retrieval-Augmented Generation.

Solves: Hallucination + Knowledge Cutoff
Model answers questions grounded in documents you provide — not from (possibly stale or wrong) memorized weights.
How it works
Give the model real documents in its context window before answering. Model weights unchanged. The knowledge lives in your data store, not the model.
02 · PatternsSession 2
02 · Tool Use

Tool Use: Let AI Take Actions.

Solves: Can't Execute
LLM outputs text saying what function to call. Your code runs it. The model never executes anything — it only describes what should happen.
The mental model
AI is the brain. Your application is the hands. Tool use = connecting thinking to doing.
02 · PatternsSession 2
02 · Agents

Agents: AI That Plans and Acts.

Chatbot vs Agent
Chatbot: 1 question → 1 answer.

Agent: plan → act → observe → repeat until goal reached.
Agent = chatbot + hands.
The loop runs until the agent decides it's done — or until it hits a limit you set.

02 · Decision Guide

When to use what?

If you need… Consider Note
Answers from internal documents RAG First thing to try for knowledge-grounded Q&A
Call APIs or query databases Tool Use / Agents Model describes what to call; your code runs it
Model to speak in company voice Fine-tuning Try prompts and RAG first — fine-tuning is expensive
Filter harmful or off-topic output Guardrails Input + output filtering layer
Multi-step reasoning and planning Agents (ReAct) More complex; harder to debug and control
If if/else is enough, don't use AI.

03 · QA Prep

QA Survival Kit — Questions 1–5.

Q1
What is RAG?
Retrieve relevant docs, put in context, let LLM answer grounded in real info.
Q2
RAG vs fine-tuning?
RAG: docs in context, weights unchanged. Fine-tune: actually change model weights.
Q3
Why does AI hallucinate?
No source of truth. Predicts probable tokens, not necessarily true ones.
Q4
What is an embedding?
Vector representation of meaning. Similar concepts tend to have similar vectors.
Q5
What is a vector database?
DB optimized for similarity search on embeddings — not SQL-style queries.
03 · Theory to ProjectSession 2
03 · QA Prep

QA Survival Kit — Questions 6–10.

Q6
Bedrock KB vs Bedrock Agents?
KB = managed RAG pipeline. Agent = managed tool use + orchestration.
Q7
What is temperature?
Controls sampling randomness. Low = deterministic, high = creative / less predictable.
Q8
What is context window?
Max tokens the model can see at once. Everything competes for this space.
Q9
What is prompt engineering?
Crafting inputs to get better outputs without changing the model itself.
Q10
Why not just fine-tune for everything?
Expensive, needs labeled data, can degrade other capabilities. Try prompts and RAG first.

Evaluation and Trust.

How do you know if your AI system is working? The uncomfortable answer.

04 · Evaluation

How do you know your AI works?

Manual Review
Read the outputs.
Check accuracy. Find failure modes. The most reliable signal — but doesn't scale beyond a few hundred samples.
Automated Eval
LLM-as-judge, metrics.
BLEU, ROUGE, similarity scores. LLM grades other LLM outputs. Scales, but introduces its own biases.
User Feedback
Real signal from real use.
Thumbs up/down, usage patterns, support tickets. Noisy but it reflects what actually matters.
Nothing replaces reading the output. But scale needs automation.
04 · EvaluationSession 2
04 · Iteration

Prompt Iteration Cycle.

Like debugging code.
Your first prompt is almost always suboptimal. Iteration is the work, not the exception. Each cycle teaches you something about where the model breaks.
No feedback loop = guessing blind.
Define what "good" looks like before running the model, not after. Otherwise you're just changing prompts randomly.

AI Safety & Responsible AI.

Bias, misuse, guardrails, and the questions that don't have clean answers yet.

05 · Bias

Bias In = Bias Out.

Selection Bias
Training data doesn't represent the full population you're trying to serve.
Label Bias
Human annotators bring their own biases into the labels used to train the model.
Representation Bias
Some groups are underrepresented in training data — model performs worse on them.
05 · Responsible AISession 2
05 · Safety

Safety & Alignment.

Jailbreaking
Tricking the model to bypass safety.
Prompt injection, social engineering, role-play attacks. "Pretend you have no restrictions…" Models are not perfectly robust against this.
RLHF
Reinforcement Learning from Human Feedback.
Human raters score model outputs. Model fine-tuned to produce outputs humans rate highly. Makes models more helpful and less harmful — imperfect, but significant.
Red Teaming
Intentionally attacking your own system.
Before deployment, hire or task people to break the model. Find failure modes before users do. Standard practice at major AI labs.
05 · Responsible AISession 2
05 · Practice

Responsible AI in Practice.

Principle What it means in practice
Transparency Tell users when they're talking to AI. Don't pretend it's a human.
Human-in-the-loop Critical decisions (medical, legal, financial) need human review before action.
Data Privacy Don't send PII to external models without user consent. Know your data residency requirements.
Monitoring Watch for drift, bias, and failures post-deployment. Models don't stay accurate forever.
Accountability When AI is wrong, who is responsible? Answer this before deployment, not after an incident.
05 · Responsible AISession 2
05 · Before You Build

5 questions before building AI into anything.

01
Does this problem actually need AI?
Would if/else or a database query solve it? Start simple.
02
What happens when it's wrong?
Wrong in a recommendation vs wrong in a medical diagnosis are very different.
03
Do we have the data?
Enough, clean, representative. No data plan = no AI plan.
04
Can we evaluate the output?
If you can't measure quality, you can't improve it. Define success criteria first.
05
Who is responsible?
When it makes a mistake, who owns it? Don't ship AI without an accountability plan.

RecapSession 2
Core Message

What to take away.

LLMs have 4 fundamental limitations.
Hallucination, knowledge cutoff, no memory, can't execute. Understanding these is the prerequisite for everything.
Every pattern in GenAI is a workaround.
RAG, Tool Use, Agents, Context Engineering — each exists because the model itself can't solve the problem alone.
Pick the right pattern for the problem.
RAG for knowledge, Tool Use to act, Agents to plan, Fine-tuning for voice, Guardrails to protect. Don't use all of them just because you can.
Building AI comes with real responsibility.
Bias, fairness, accountability, transparency. These aren't optional extras — they're part of the job.

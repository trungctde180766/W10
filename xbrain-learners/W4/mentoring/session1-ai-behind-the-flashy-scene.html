Everything is probability.

"Toi dang o ___"
The model samples from a probability distribution.
This is how an LLM generates text — it samples from a probability distribution over the next token.

No human-like understanding. No deliberate reasoning. Just: which word is statistically most likely to come next?

02 · Self-tuning

Backpropagation: How the Machine Tunes Itself.

Gradient = "how much to adjust each knob, and in which direction." Repeat thousands of times → model gets better.
02 · How Machines LearnSession 1
02 · Intuition

Gradient Descent = Ball Rolling Downhill.

"Lowest reachable point" = lowest training error. But too low on training data may mean overfitting (next slide).
Each roll = 1 training step.
Learning rate = how big each step is. Too big → overshoot. Too small → stuck forever.
02 · How Machines LearnSession 1
02 · Failure mode

Overfitting = Rote Memorization.

Training Data
99%
Exam 1, 2, 3 — near-perfect
New Data (Test)
58%
Exam A, B, C — fails badly
Memorize 100 past exams → 10/10. New exam → 2/10. Gap = OVERFITTING.

04 · Black BoxSession 1
04 · Control

Temperature: Controlling Randomness.

temp = 0
Always picks the top token. Deterministic. Same input → same output. Good for code, math, factual answers.
temp = 0.7
Balanced. Some creativity, still coherent. Good for writing, brainstorming.
temp = 1.0+
Flatter distribution. More randomness → more surprising (or incoherent). In practice most APIs cap at 1.0–2.0; values above 1 are rarely used in production.
Temperature scales the logits before softmax. T < 1 sharpens the distribution (concentrates on top tokens). T > 1 flattens it (more uniform). The model's weights don't change — only the sampling distribution does.

05 · Meaning as math

Embedding: Make Meaning Computable.

This is how AI "understands" meaning.
Every word becomes a vector (list of numbers). Similar concepts get similar vectors.

Everything in AI that processes text relies on embeddings.

05 · LanguageSession 1
05 · Attention

Attention: How AI Reads Context.

When reading "it", pay more attention to "cat".
This is called Attention.
The mechanism that lets a model decide which words matter most for predicting the next one.
Transformer enables this.
Processes all tokens in parallel, with attention scores between every pair.
Every modern LLM uses it.
GPT, Claude, Gemini, Llama — all Transformers.

05 · Scale

Pre-training: Reading the Internet.

Training data
~TB
of text from books, web, code, papers
Task
Predict Next Token
Billions of times
Result
World model?
Grammar, facts, reasoning — all emerge from prediction.
Pre-training gives the base model. Fine-tuning + RLHF makes it helpful and safe. GPT-style models predict the next token. BERT-style models fill in masked tokens (different task, same idea: learn language patterns from raw text).
05 · LanguageSession 1
05 · Memory

Context Window: The Model's Memory Limit.

Model only "sees" what's inside the frame.
Everything competes for this limited space. Forgotten content = doesn't exist for the model.
200 pages → "forgets" page 50?
The model sees all tokens in the window — but attention weights distribute unevenly. Content in the middle of long contexts gets less attention ('lost-in-the-middle' effect).

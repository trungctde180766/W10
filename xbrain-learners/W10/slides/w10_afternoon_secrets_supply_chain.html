Secrets Rotation

Secret plain text trong git?

AWS Secrets Manager + External Secrets Operator.
Rotate < 60s, không restart pod.

Problem

Secret hard-code trong manifest

DB password commit thẳng vào git. Ai clone repo = ai có password.

Password plain text trong git

DB_PASSWORD: "prod@123" commit vào repo. Git history giữ mãi. Leak 1 lần = toàn bộ lịch sử lộ.

Rotate cần restart pod

Đổi password trong Secret → pod phải restart mới thấy. Downtime 30s mỗi lần rotate. SLO vi phạm.

Không biết khi nào rotate

Password dùng 2 năm chưa đổi. Không có policy tự động rotate. Compliance audit fail.

ConfigMap dùng làm Secret

Dev dùng ConfigMap chứa password vì "tiện hơn Secret". Base64 không phải encryption.

Vấn đề gốc:
• K8s Secret chỉ là base64, không encrypt
• Rotate = restart pod

01 — Secrets Rotation
Lợi ích

ESO — Rotate < 60s, không restart pod

2 điểm mạnh: tự động reload + configurable refresh.

No restart needed

Pod mount Secret qua volume. ESO update Secret → volume tự reload. App đọc file mới.

Kết quả: Không cần restart pod. Zero downtime khi rotate.
refreshInterval

ESO poll AWS Secrets Manager mỗi refreshInterval (vd 60s). Thấy thay đổi → sync ngay.

W10 target: < 60s từ AWS rotate đến pod thấy mật khẩu mới.

Supply Chain Security

Image không rõ nguồn gốc?

Trivy scan trong CI + Cosign sign image + Admission verify signature.

Problem

Deploy image không biết ai build

CI không scan CVE. Cluster pull image từ registry lạ. Không verify signature.

CI không scan CVE

Build xong push luôn. Không biết image có CVE HIGH/CRITICAL. Production chạy vulnerability 6 tháng.

Image không sign

Ai cũng push image lên registry. Cluster pull về không biết ai build, build lúc nào, có tamper không.

Cluster không verify

Admission không check signature. Dev deploy image từ laptop cá nhân. Supply chain attack dễ như ăn kẹo.

CVE vendor chưa fix

Image có CVE HIGH nhưng vendor chưa release patch. CI fail → block deploy. Không có exception process.

Giải pháp: Trivy scan trong CI (fail on HIGH/CRITICAL) + Cosign sign image + Admission verify signature trước khi pull.

02 — Supply Chain Security
Kết quả

Pass scan vs Fail scan

2 kịch bản: image clean hoặc có CVE HIGH/CRITICAL.

Pass scan
Image không có CVE HIGH/CRITICAL
Flow:
1. Trivy scan pass ✓
2. Tiếp tục bước 2 (Cosign sign)
3. Push image lên registry
4. Deploy được vào cluster
Fail scan
Image có CVE HIGH hoặc CRITICAL
Flow:
1. Trivy scan fail ✗
2. CI pipeline stop ngay
3. Không push image
4. Cần fix CVE hoặc viết exception ADR

Lab 2.1 · ESO · Update App

Update app để test DB secret

app/app.py - Đọc password từ mounted secret volume.

# app/app.py - Setup + Helper functions import os import random from flask import Flask, jsonify from prometheus_flask_exporter import PrometheusMetrics app = Flask(__name__) PrometheusMetrics(app) # /metrics VERSION = os.getenv("VERSION", "v1") DB_PASSWORD_PATH = os.getenv("DB_PASSWORD_PATH", "/secrets/password") def get_db_password(): """Read DB password from volume""" try: with open(DB_PASSWORD_PATH, 'r') as f: return f.read().strip() except FileNotFoundError: return "SECRET_NOT_FOUND" except Exception as e: return f"ERROR: {str(e)}" def should_simulate_error(): """10% fail rate (1/10 requests)""" return random.random() < 0.1
# Routes @app.get("/") def index(): if should_simulate_error(): return jsonify( error="simulated_error", version=VERSION ), 500 db_password = get_db_password() db_status = "connected" \ if db_password != "SECRET_NOT_FOUND" \ else "disconnected" return jsonify( ok=True, version=VERSION, db_status=db_status, db_password_loaded=db_password != "SECRET_NOT_FOUND" ) @app.get("/healthz") def healthz(): return "ok", 200 @app.get("/db-secret") def db_secret(): """Debug endpoint""" password = get_db_password() return jsonify( password_path=DB_PASSWORD_PATH, password_found=password != "SECRET_NOT_FOUND", password_preview=password[:5] + "..." if len(password) > 5 else password )

# .github/workflows/build.yml name: Build and Scan on: push: { branches: [main, demo] } jobs: build: runs-on: ubuntu-latest steps: - name: Checkout uses: actions/checkout@v4 - name: Login to GHCR uses: docker/login-action@v3 with: registry: ghcr.io username: ${{ github.actor }} password: ${{ secrets.GITHUB_TOKEN }} - name: Set image name id: image run: echo "name=$(echo ghcr.io/${{ github.repository }} | tr '[:upper:]' '[:lower:]')" >> $GITHUB_OUTPUT - name: Build image run: docker build -t ${{ steps.image.outputs.name }}:${{ github.sha }} app/

Note: Convert repository name to lowercase (GHCR requirement). GITHUB_TOKEN tự động có sẵn.
